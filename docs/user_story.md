# User Stories

## 사용자 스토리 및 인수 조건 (User Stories & Acceptance Criteria)

### 1. 인증 (Authentication)

#### 1.1 회원가입
- **Story**: 사용자는 서비스 이용을 위해 이메일, 비밀번호, 닉네임으로 회원가입을 할 수 있다.
- **Acceptance Criteria**:
    - [ ] 이메일 형식이 올바르지 않으면 에러 메시지를 표시해야 한다.
    - [ ] 비밀번호는 4자 이상이어야 한다.
    - [ ] 회원가입 성공 시 자동으로 로그인 처리가 되거나 로그인 페이지로 이동해야 한다.

#### 1.2 로그인/로그아웃
- **Story**: 사용자는 이메일과 비밀번호로 로그인하고, 원할 때 로그아웃할 수 있다.
- **Acceptance Criteria**:
    - [ ] 로그인 성공 시 Access Token과 Refresh Token을 발급받아야 한다.
    - [ ] 앱을 새로고침해도 로그인 상태가 유지되어야 한다 (Silent Refresh).
    - [ ] 로그아웃 시 토큰이 만료되고 로그인 페이지로 이동해야 한다.

### 2. 성적 분석 (Grade Analysis)

#### 2.1 이미지 업로드 및 분석
- **Story**: 사용자는 성적표 스크린샷을 업로드하여 AI 분석 결과를 받을 수 있다.
- **Acceptance Criteria**:
    - [ ] 5MB 이하의 이미지 파일(JPG, PNG 등)만 업로드 가능해야 한다.
    - [ ] 업로드 중에는 로딩 인디케이터가 표시되어야 한다.
    - [ ] 분석 실패 시 적절한 에러 메시지(토스트)를 보여줘야 한다.

#### 2.2 데이터 검증 및 시각화
- **Story**: 사용자는 분석된 데이터를 직접 수정하고, 최종적으로 시각화된 리포트를 볼 수 있다.
- **Acceptance Criteria**:
    - [ ] 분석 결과(과목명, 학점, 성적)가 테이블 형태로 표시되어야 한다.
    - [ ] 사용자는 테이블의 내용을 직접 수정할 수 있어야 한다.
    - [ ] '분석 완료' 버튼을 누르면 평점 추이 그래프와 이수 구분 차트가 렌더링되어야 한다.

### 3. 게시판 (Board)

#### 3.1 게시글 관리
- **Story**: 사용자는 게시글을 작성, 조회, 수정, 삭제할 수 있다.
- **Acceptance Criteria**:
    - [ ] 로그인한 사용자만 글을 작성할 수 있다.
    - [ ] 본인이 작성한 글만 수정 및 삭제 버튼이 보여야 한다.
    - [ ] 게시글 목록은 페이지네이션 처리되어야 한다.

#### 3.2 댓글 소통
- **Story**: 사용자는 게시글에 댓글을 남겨 소통할 수 있다.
- **Acceptance Criteria**:
    - [ ] 게시글 상세 페이지 하단에 댓글 목록이 표시되어야 한다.
    - [ ] 댓글 작성 후 즉시 목록에 반영되어야 한다.

### 4. 프로필 관리 (Profile Management)

#### 4.1 내 정보 조회
- **Story**: 사용자는 자신의 프로필 페이지에서 닉네임, 이메일, 활동 내역(작성한 글/댓글)을 확인할 수 있다.
- **Acceptance Criteria**:
    - [ ] 프로필 페이지 진입 시 내 정보가 로딩되어야 한다.
    - [ ] 내가 작성한 게시글과 댓글 목록을 탭으로 구분하여 볼 수 있어야 한다.

#### 4.2 닉네임 수정
- **Story**: 사용자는 자신의 닉네임을 변경할 수 있다.
- **Acceptance Criteria**:
    - [ ] 닉네임 수정 버튼을 누르면 입력 필드가 활성화되어야 한다.
    - [ ] 수정 완료 시 서버에 반영되고, 화면에 즉시 업데이트되어야 한다.
    - [ ] 기존과 동일한 닉네임으로는 변경할 수 없다.

### 5. 서비스 접근 (Access & Navigation)

#### 5.1 서비스 소개 및 진입
- **Story**: 비로그인 사용자는 랜딩 페이지를 통해 서비스의 주요 기능을 파악하고 시작할 수 있다.
- **Acceptance Criteria**:
    - [ ] 루트 경로(`/`) 접속 시 랜딩 페이지가 보여야 한다.
    - [ ] '시작하기' 버튼을 누르면 로그인 상태에 따라 메인 페이지 또는 로그인 페이지로 이동해야 한다.
